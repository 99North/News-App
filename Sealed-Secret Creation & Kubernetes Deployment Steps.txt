=>Step by Step guide to install & use SealedSecret in kubernetes 
kubectl apply -f https://github.com/bitnami-labs/sealed-secrets/releases/download/v0.27.0/controller.yaml
kubectl get pods -n kube-system | grep sealed
wget https://github.com/bitnami-labs/sealed-secrets/releases/download/v0.32.2/kubeseal-0.32.2-linux-arm64.tar.gz
tar -xvzf kubeseal-0.32.2-linux-arm64.tar.gz kubeseal
sudo install -m 755 kubeseal /usr/local/bin/kubeseal
kubeseal --version
ls -l /etc/rancher/k3s/k3s.yaml
chmod 744 /etc/rancher/k3s/k3s.yaml
export KUBECONFIG=/etc/rancher/k3s/k3s.yaml

Now run these two command to make this sealed-secret extraction permanent
echo "export KUBECONFIG=/etc/rancher/k3s/k3s.yaml" >> ~/.bashrc
source ~/.bashrc

create a namesapce name news-app:  kubectl create namespace news-app
change the current namespace to it:  kubectl config set-context --current --namespace=news-app
check which namespace you are in:   kubectl config view --minify | grep namespace


Now create your backend-secrete.yaml file & postgres-secret.yaml file and paste the secret deatils as in gihub repo and save it
Now run this to convert the plane secret file to sealed secret version:   kubeseal --format=yaml < backend-secret.yaml > backend-sealed.yaml  &  kubeseal --format=yaml < postgres-secret.yaml > postgres-sealed.yaml

To see the secret is created or not:   kubectl get sealedsecrets -n news-app
Now apply all the deployment file one by one like bellow
kubectl apply -f frontend-deployment.yaml
kubectl apply -f backend-deployment.yaml
kubectl apply -f postgres-stateful-deployment.yaml


Now run this command to add user-credential into the users database directly
curl -X POST http://52.66.27.150:32001/auth/register \
-H "Content-Type: application/json" \
-d '{ "username": "debi", "email": "debi@gmail.com", "password": "debipass", "firstName": "Debi", "lastName": "Prasad" }'


To verify run these commands one by one:
kubectl exec -it postgres-0 -n news-app -- bash   :to enter inside the db-pod
psql -U debi -d newsdb     :check username & database name exist or not. if exist it show this:   newsdb=#
\l     :to list all databases inside the database-server
\dt   :to list all database tables
\d  users:   to list all the users inside it
SELECT * FROM users;      :Check table data
\q     :to quite form the database
\exit    : to exit from the pod

OR
You can run this one line command to check directly:   kubectl exec -it postgres-0 -n news-app -- bash -c "psql -U debi -d newsdb -c '\dt'"


NOTE:  To check if the init.sql script has run inside the container or not run these bellow command
kubectl exec -it postgres-0 -n news-app -- bash 
ls /docker-entrypoint-initdb.d/

To check if the script was executed: ls /var/lib/postgresql/data
